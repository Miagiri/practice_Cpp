## Ключевые шаги диалога

### 1. Постановка L5
- Агент описал: C++ через `std::unordered_map`, C — ручная реализация с цепочками (chaining).

### 2. Объяснение хеш-таблицы на C
- Две структуры: Node (key, value, next) и HashTable (buckets, size).
- Три операции: hash (key % size), insert (в начало цепочки, O(1)), search (проход по цепочке).
- Наводящие вопросы про размер таблицы.

### 3. Дискуссия: почему простые числа для размера таблицы
- Ирина правильно ответила про TABLE_SIZE=10 (длинные цепочки).
- Агент объяснил на примерах: TABLE_SIZE=8 (степень двойки) берёт только младшие биты, закономерности данных ведут к коллизиям. TABLE_SIZE=7 (простое) размазывает равномерно.
- Ирина верно ответила: при вставке пробы не считаем.

### 4. C++ версия (search_l5.cpp)
- Ирина написала код, агент нашёл 3 замечания:
  1. `rand()` вместо `dis(gen)` для выбора ключа — критично.
  2. `probes = 1` всегда — вводит в заблуждение. Вместо этого — `bucket_count()`, `load_factor()`, `bucket_size()`.
  3. Двойной поиск при вставке (`find` + `operator[]`) → `try_emplace`.
- Ирина исправила все три. Код принят.

### 5. C версия (search_l5.c)
