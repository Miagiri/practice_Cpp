_Thinking:_

The user wants to understand the purpose of Debug+ASan mode before making changes to build.bat.

### `/permissive-`

Стандартный режим компилятора MSVC пропускает некоторые нестандартные конструкции (расширения Microsoft, C-привычки). `/permissive-` включает **строгий режим соответствия стандарту C++**. Пример: без него двухфазный поиск имён в шаблонах не работает, некоторые неявные приведения типов проходят молча. С ним — компилятор ловит код, который не перенесётся на GCC/Clang.

### `/Od /MDd /fsanitize=address` — Debug + ASan

- **`/Od`** — без оптимизаций. Код выполняется 1:1 как написан, отладчик идёт по строкам без прыжков.
- **`/MDd`** — отладочная версия CRT (C Runtime). Включает проверки итераторов STL (`_ITERATOR_DEBUG_LEVEL=2`): выход за границы вектора, невалидный итератор — ловятся на месте.
- **`/fsanitize=address`** — AddressSanitizer (ASan). Ловит use-after-free, buffer overflow, stack overflow — баги, которые без санитайзера могут молча портить память и всплыть крашем через час в другом месте.

### Зачем сейчас

До этого ты собирала с `/W4 /analyze /Zi /MD` — это Release-режим с отладочной информацией. Он быстрее, но не ловит ошибки памяти. ASan и отладочный CRT замедляют программу, но если в коде есть выход за границу — ты узнаешь об этом **в момент ошибки**, а не через неделю гадания над дампом.
