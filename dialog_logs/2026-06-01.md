## Что делает `find_if`

Представь, что у тебя есть коробка с пронумерованными ячейками (это вектор). Ты хочешь найти первую ячейку, в которой лежит число 9.

```

  в каждой смотрим: это 9?

    нет → идём дальше
```

`std::find_if` — это **готовая функция из STL**, которая делает ровно то же самое. Ты говоришь ей: «вот контейнер, вот условие» — и она возвращает **указатель на найденный элемент** (или «конец контейнера», если не нашла).

---

## Два варианта: старый и новый

**Старый `std::find_if`** (C++11):
```cpp
auto it = std::find_if(data.begin(), data.end(), условие);
```
Ты передаёшь начало и конец — два итератора. Это как сказать «ищи от третьего шкафа до десятого».

**Новый `std::ranges::find_if`** (C++20):
```cpp
auto it = std::ranges::find_if(data, условие);
```
Ты передаёшь **весь контейнер целиком**. Короче, меньше шансов ошибиться (не перепутать begin/end двух разных контейнеров).

---

## Как задать условие

Условие — это **лямбда** (маленькая функция прямо в строке):

```cpp
auto it = std::ranges::find_if(data, [](int x) { return x == 9; });
```

`it` — это итератор. Если нашли — `it` указывает на ячейку с 9. Если не нашли — `it == data.end()`.

---

## Но у нас есть подвох: подсчёт сравнений

В твоём коде ты считаешь, сколько раз программа заглянула в ячейку (`comparisons`). Лямбда может это делать, если **захватит переменную по ссылке**:

```cpp
size_t comparisons = 0;
auto it = std::ranges::find_if(data, [&](int x) {
    ++comparisons;
    return x == 9;
});
```

`[&]` означает «лямбда видит переменную `comparisons` и может её менять». Каждый раз, когда `find_if` проверяет ячейку, лямбда дёргает счётчик.

**`[]` — блок захвата (capture)**

Говорит: «какие переменные из внешнего мира лямбда видит».

То есть: `find_if` берёт первый элемент `5`, вызывает лямбду с `x = 5`. Потом берёт следующий `2`, вызывает с `x = 2`. И так далее.

- `не 0` = что-то пошло не так (ошибка компиляции, файл не найден, и т.д.)

## Пункт 2: `/Fo:` — «положи готовый файл сюда»
