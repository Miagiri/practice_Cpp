### Что такое `.bat`-файл

Обычный текстовый файл, в котором каждая строка — команда, как в консоли. CMD выполняет их последовательно. Расширение `.bat` говорит системе «запусти это в CMD».

### Что он должен делать для твоего проекта

```
1. Настроить окружение (MSVC, cl.exe, link.exe)
2. Скомпилировать C-версию  (cl /c → .obj → link → .exe)
3. Скомпилировать C++-версию (cl /c → .obj → link → .exe)
4. Сообщить «собрано» или «ошибка»
```

### Загвоздка

`setup_env.ps1` написан на PowerShell. `.bat` работает в CMD. Если CMD позовёт PowerShell, переменные окружения (`PATH`, `INCLUDE`, `LIB`) умрут вместе с дочерним процессом — и следующий `cl` в `.bat` не найдётся.

**Два решения:**
- **А) Писать на `.ps1`** (PowerShell) — проще и роднее для твоего окружения, никаких проблем с переменными
- **Б) `.bat`, который целиком дёргает PowerShell** — одной строкой настраивает окружение и собирает всё внутри одного powershell-вызова

### Рекомендую вариант А (`build.ps1`)

Структура:

```powershell
# build.ps1 — чистая сборка задачи 7.1

. .\setup_env.ps1                          # включить MSVC

# C-версия
cl /c /W4 /analyze /Zi /MD linear-collections\c_version\search_l1.c
link search_l1.obj /OUT:linear-collections\c_version\search_l1.exe /DEBUG

# C++-версия
cl /c /W4 /analyze /Zi /MD /EHsc /std:c++latest linear-collections\cpp_version\search_l1.cpp
link search_l1.obj /OUT:linear-collections\cpp_version\search_l1.exe /DEBUG

Write-Host "Build done."
```
